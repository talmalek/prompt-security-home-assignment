# Skill style guide

This guide defines the standard structure for skills in this repository.

## Frontmatter

Keep frontmatter minimal:

```yaml
---
name: <skill-id>
description: Use this skill when ...; triggers include ...
---
```

## Progressive disclosure

Keep `SKILL.md` concise; move long content to `references/`.

## Language

Skills are written in English. Align section titles with the automation-tester skill for consistency.
